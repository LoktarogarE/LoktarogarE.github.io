OS Chapter 3

3.1 内存管理基础

多级页表中，顶级页表最多只能有 1 个页面。

虚拟内存管理中，FIFO 算法还会产生所分配的物理块数增大而页故障数不减反增的异常现象，称为 Belady 异常。

在虚拟内存管理中，地址变换机构将逻辑地址变换为物理地址，形成该逻辑地址的阶段是链接 ( 编译后的程序需要经过链接才能装载，而链接后形成的目标程序中的地址也就是逻辑地址 )

虚拟内存管理需要由相关的硬件和软件支持，有请求分页页表机制、缺页中断机构、地址变换机构等。

在存储管理中，采用覆盖和交换技术的目的是节省主存空间。

分区分配内存管理方式的主要保护措施是界地址保护。

存储管理方案中，单一连续存储管理可采用覆盖技术。

静态装入是指在编程阶段就把物理地址计算好。

可重定位是指在装入时把逻辑地址转换成物理地址，但装入后不能改变。

动态重定位是指在执行时再决定装入的地址并装入，装入后可能会换出，所以同一个模块在内存中的物理地址是可能改变的。

动态重定位是指在作业运行过程中执行到一条访存指令时，再把逻辑地址转换为主存中的物理地址，实际中是通过硬件地址转换机制实现的。

多进程在主存中彼此互不干扰的环境下运行，操作系统时通过内存保护来实现的。

系统提供给用户的物理地址空间为总空间大小减去页表或段表的长度。

在重定位存储管理方式，应在整个系统中设置一个重定位寄存器。

分段存储管理有利于程序的动态链接。

可重入程序主要是通过共享同一块存储空间的，或通过动态链接的方式将所需的程序段映射到相关程序中去，其最大的优点是减少了对程序段的调入/调出，因此减少了对换数量。

对外存对换区的管理以提高换入、换出速度为目标。

引入段式存储管理方式，主要是为了满足用户的下列要求：方便编程、分段共享、分段保护、动态链接、动态增长。

存储管理的目的是方便用户和提高内存利用率。

对主存储器的访问，以字节或字为单位。

段页式存储管理用分段方法来分配和管理用户地址空间，用分页方法来管理物理存储空间。

最佳适应算法是最容易产生最多的内存碎片。

3.2 虚拟技术存储管理

在请求分页存储系统中，若采用 FIFO 页面淘汰算法，则当可分配的页帧数增加时，缺页中断次数可能增加也可能减少。

虚拟存储器的最大容量是由计算机的地址结构决定的，与主存容量和外存容量没有必然的联系。

导致 LRU 算法实现起来耗费高的原因是需要对所有的页进行排序。

请求分页存储管理的主要特点是扩充了内存。

在请求分页存储管理的页表中增加了若干项信息，其中修改位和访问位供置换算法参考。

能加快虚实地址转换的是：增大 TLB 容量、让页表常驻内存。

在任何一个时刻 $t$，都存在一个集合，它包含所有最近 $k$ 次内存访问所访问过的页面，这个集合 $w(k,t)$ 就是工作集 ( ⚠️：工作集不是要最近的六个不同的访存操作，可以是相同的，就是 $k$ 次 )

影响请求分页系统有效 ( 平均 ) 访存时间的事：缺页率、磁盘读写时间、内存访问时间、执行缺页处理程序的 CPU 时间。
